# Заявка на Итоговую Выпускную Работу (ИВР)

**Тема:** Разработка интеллектуального мобильного приложения для структурирования учебных материалов «AudioLeash».
**Автор:** Тарасов Григорий.
10и5

---

### 👥 Целевая аудитория / заказчик
* **Первичная аудитория:** Учащиеся Лицея НИУ ВШЭ и студенты вузов, посещающие лекционные занятия.
* **Вторичная аудитория:** Люди, посещающие профессиональные конференции, вебинары или интервью, которым необходимо быстро фиксировать и структурировать большой объем аудиоинформации.

### 🚩 Проблемное поле
Существующие методы фиксации лекций неэффективны:
1. **Диктофоны** создают «линейные» файлы, в которых невозможно быстро найти нужную информацию без полного переслушивания.
2. **Ручные конспекты** часто фрагментарны: ученик отвлекается на запись формулы и пропускает объяснение учителя.
3. **Отсутствие связи** между аудиозаписью, визуальными данными (фото доски) и текстовыми заметками.

### 🎯 Зачем нужен проект?
Проект создает «интеллектуальный слой» над аудиозаписью, решая проблемы:
* **Потери времени:** авто-транскрибация и суммаризация позволяют ознакомиться с лекцией за 5 минут.
* **Фрагментарности знаний:** фото и заметки привязаны к таймкодам в едином пространстве.
* **Когнитивной перегрузки:** AI-тьютор помогает разъяснить сложные моменты сразу после занятия.

### 📦 Образ продукта
Мобильное приложение для Android со следующими характеристиками:
* **Маркеры в реальном времени:** Кнопки создания таймстемпов («Важно», «Определение», «Пример»).
* **Фото-привязка:** Автоматическое прикрепление фото к секунде записи.
* **Интерактивный плеер:** Навигация по таймлайну через иконки-маркеры.
* **Блок AI-аналитики:** Полная расшифровка (STT), краткий конспект (Summary) и чат-бот по контексту записи.
* **Экспорт:** Генерация отчетов в формате PDF/Markdown.

### 🤝 Взаимодействие
* **Куратор ИВР:** Консультации по архитектуре и документации.
* **Лицеисты (Тестировщики):** Сбор фидбека по интерфейсу (3-5 человек).
* **Преподаватель по Android:** Техническое менторство.
* **Преподаватели Лицея:** Согласование записи материалов для тестов.

### 🛠 Ресурсы
* **Технические:** Смартфон Android (10+), PC с Android Studio.
* **Программные:** API OpenAI/Google Cloud, библиотека Vosk.
* **Информационные:** Документация Kotlin, Jetpack Compose, Room.

### 🎓 Образовательные цели
В процессе реализации мне предстоит освоить:
1. **Android Development:** Jetpack Compose, управление мультимедиа (аудио/камера).
2. **Архитектура ПО:** Паттерн MVVM.
3. **AI Integration:** Работа с REST API, Prompt-инжиниринг, NLP.
4. **БД:** Реляционные связи в Room.

### 📅 Поэтапное планирование
1. **Этап 1 (MVP):** Запись аудио, сохранение файла, список записей.
2. **Этап 2 (Markers):** Таблица Markers в БД, визуализация на Seekbar.
3. **Этап 3 (Camera):** Съемка внутри приложения через CameraX, связь с recordingId.
4. **Этап 4 (Transcription):** Интеграция Vosk (STT).
5. **Этап 5 (AI & Export):** Суммаризация, ИИ-тьютор, генерация PDF-отчетов.

### ⚠️ Риски
* **Энергопотребление:** Решается оптимизацией фоновых процессов.
* **Шум в аудитории:** Решается фильтрацией звука или использованием устойчивых моделей (Whisper).
* **Стоимость API:** Использование бесплатных альтернатив (Vosk) в качестве базового функционала.
